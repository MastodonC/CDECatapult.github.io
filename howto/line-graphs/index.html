Line graphs

Line graphs are like analog bar charts. Like bar charts they work when you have two variables - but rather than discrete values, the data is continuous like the progression through time, or a temperature rising and falling.

As well as displaying known points, a line graph allows us to look at the values between - and beyond - the points.

This tutorial will give you a few pointers on making line graphs work and how to keep them looking smart.

When to use

The value of a line graph is in seeing the change of one continuous variable relative to another. For example, plotting the velocity of an object over time.

We can only measure the velocity at specific points in time, but we know that velocity is a continuous variable so we can connect the dots and reliably demonstrate what happens in the gaps - this is called interpolation.

As well as showing us what lies in the gaps, line graphs can also be used to project trends into the past or future. After the data points have been collected, a best fit line is created and extended in both directions. The validity of any projection is in no way guaranteed by the use of a line chart, but the chart can be used as a visual aid.

Some example data + coding up

We're creating these examples using HTML, CSS and a JavaScript data visualisation library called D3.js.

You can use these examples as a base for your own visualisations - the code is fully commented up and you can download them as a zip below. If you want to explore D3.js further you can find many examples online.

            
<!DOCTYPE html>
<meta charset="utf-8">
<style>

.bar {
  fill: steelblue;
}

.bar:hover {
  fill: brown;
}

.axis {
  font: 10px sans-serif;
}

.axis path,
.axis line {
  fill: none;
  stroke: #000;
  shape-rendering: crispEdges;
}

.x.axis path {
  display: none;
}

</style>
<body>
<script src="http://d3js.org/d3.v3.min.js"></script>
<script>

var margin = {top: 20, right: 20, bottom: 30, left: 40},
    width = 960 - margin.left - margin.right,
    height = 500 - margin.top - margin.bottom;

var x = d3.scale.ordinal()
    .rangeRoundBands([0, width], .1);

var y = d3.scale.linear()
    .range([height, 0]);

var xAxis = d3.svg.axis()
    .scale(x)
    .orient("bottom");

var yAxis = d3.svg.axis()
    .scale(y)
    .orient("left")
    .ticks(10, "%");

var svg = d3.select("body").append("svg")
    .attr("width", width + margin.left + margin.right)
    .attr("height", height + margin.top + margin.bottom)
  .append("g")
    .attr("transform", "translate(" + margin.left + "," + margin.top + ")");

d3.tsv("data.tsv", type, function(error, data) {
  x.domain(data.map(function(d) { return d.letter; }));
  y.domain([0, d3.max(data, function(d) { return d.frequency; })]);

  svg.append("g")
      .attr("class", "x axis")
      .attr("transform", "translate(0," + height + ")")
      .call(xAxis);

  svg.append("g")
      .attr("class", "y axis")
      .call(yAxis)
    .append("text")
      .attr("transform", "rotate(-90)")
      .attr("y", 6)
      .attr("dy", ".71em")
      .style("text-anchor", "end")
      .text("Frequency");

  svg.selectAll(".bar")
      .data(data)
    .enter().append("rect")
      .attr("class", "bar")
      .attr("x", function(d) { return x(d.letter); })
      .attr("width", x.rangeBand())
      .attr("y", function(d) { return y(d.frequency); })
      .attr("height", function(d) { return height - y(d.frequency); });

});

function type(d) {
  d.frequency = +d.frequency;
  return d;
}

</script>

3

That's better!

Here is the same graph with some visual improvements. And here's what they are:

  • Remove Chartjunk - chart borders, grid lines and unnecessary tick mark have gone, leaving the chart feeling less cluttered.
  • Use softer lines for the axes - strong black can look too harsh on white, by softening it a touch the axes look more considered and also allow your data to stand out.
  • Consider font sizes - a smaller font size for the axis text is more subtle and allows your data to stand out.
  • Consider line weight - make your line weight thin enough to be precise and wide enough to stand out.
